    TLDR

    Performed nmap reconnaissance to identify an nginx HTTPS service on port 443. Confirmed the Heartbleed vulnerability with the ssl-heartbleed NSE script, then used a public Python exploit to dump server memory and extract private keys plus credentials.

    Challenge Description

    This beginner-friendly TryHackMe room focuses on one of the most famous vulnerabilities in internet history — Heartbleed (CVE-2014-0160). The goal is to exploit a vulnerable HTTPS service running a flawed version of OpenSSL to leak sensitive data from the server’s memory, including private keys and credentials.

    Reconnaissance

    Before I even attempted the challenge, I had no idea what *HeartBleed* was. I decided to do some light research to familiarize myself with the vulnerability.

    > HeartBleed Official Site
    > Sean Cassidy: Diagnosis of the OpenSSL Heartbleed Bug

    In essence, Heartbleed is a critical memory disclosure vulnerability in the Heartbeat extension (RFC 6520) of OpenSSL versions 1.0.1 to 1.0.1f. It stemmed from a missing bounds check when processing heartbeat requests — the server blindly trusted the length value supplied by the client instead of using the actual size of the data received. This allowed attackers to request up to 64KB of the server’s memory per request, leaking sensitive data such as private keys and credentials.


    I started by checking for an active web server and found one running on tcp/443:

    nmap -sV -sC -p 443 10.67.75.61

    Then I confirmed the web server was running a vulnerable version:

    nmap -p 443 --script ssl-heartbleed 10.67.75.61

    During recon, I came across a GitHub repo documenting the HeartBleed exploit.

    The repo was plug-and-play, so I cloned the repo temporarily to my ~/Downloads folder and ran the script on my target.

    python2 heartbleed-exploit.py 10.67.75.61

    Exploit worked perfectly. I opened out.txt (the output file containing leaked data), and found my completion flag alongside user credentials.

    Key Takeaways


    Vulnerability-Specific Scanning Always search for targeted NSE scripts like nmap --script ssl-heartbleed during reconnaissance — version detection alone is often not enough.


    Understand the Vulnerability Heartbleed is a unique memory disclosure bug caused by a missing bounds check in OpenSSL’s Heartbeat extension. Without prior research, I would have gone in blindly trying various tactics.


    Public Exploits Work Reliable, ready-made exploit scripts (like the Python Heartbleed PoC) can be extremely effective. I should continue to accrue useful (and secure) exploits that can give me a unique edge.


    Memory Leaks Are Dangerous Prior to this, I was quite unaware of how memory leaks could impact confidentiality in a given system. Watching how easy it was for me to extract data underscored the importance of using secure versions and potentially implementing controls like encryption or secure enclaves to protect working memory.
