    TLDR:

    Reconnaissance

    I began by verifying connectivity with ping and scanning the target using nmap to identify open ports and services.

    After confirming a web server was running, I used curl to inspect the homepage and discovered the username R1ckRul3s.

    curl -v http://10.64.147.126

    I then performed directory brute-forcing with Gobuster and discovered several interesting paths, most notably robots.txt which contained the password Wubbalubbadubdub, and login.php.

    Gobuster Directory Enumeration

    Using the credentials from robots.txt, I logged into the application and gained access to the Command Panel (a restricted pseudo-CLI web shell).

    Exploitation

    Once inside the command panel, I began hunting for the three ingredients.

    First Ingredient

    Located directly in the web root as Sup3rS3cretPickl3Ingred.txt.

    Second Ingredient

    Following the clue, I checked /home/rick/ and found the file second ingredients. Since cat/less were disabled, I used strings to read it.

    Third Ingredient

    After checking privileges with sudo -l (which showed full root access), I listed the root directory and retrieved the final ingredient.

    Key Takeaways

    Always Check robots.txt
    During web reconnaissance, robots.txt frequently contains hidden paths or credentials that significantly speed up enumeration.

    Restricted Shells
    When common commands like cat, less, and more are disabled, tools like strings become invaluable for extracting readable text from files.

    Check sudo Rights Early
    Running sudo -l should be one of the first commands after gaining shell access — it can instantly reveal full system control.

    Web Command Execution
    Web shells (even limited ones) combined with poor file permissions can lead to complete server compromise very quickly.

    Systematic Enumeration
    Following clues while thoroughly exploring the filesystem (web root → user directories → root) is often enough to solve beginner boxes without complex exploits.
