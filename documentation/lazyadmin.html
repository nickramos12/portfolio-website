    Reconnaissance

    As always, I started by running NMAP, followed by searchsploit for immediate version vulnerabilities.

    I found two open ports: tcp/22 running OpenSSH 7.2p2 & tcp/80 running Apache 2.4.18. Since this version of Apache wasn't up to date, I ran searchsploit -w apache 2.4.18, which came back with an exploit involving local privilege escalation via the logrotate system.

    After further research, I found this exploit wouldn't serve me well in the immediate future, although a good vulnerability to fall back on. The exploit involves injecting code, then waiting for logrotate to automatically run (which takes place at 6:25 AM), which should escalate the user to root.

    Enumeration

    Came back with quite a few hits, but the current running version of SweetRice is still in question so I moved to running gobuster on /content to hopefully come back with more directories.

    Gobuster returned multiple hidden directories & files for /content. Immediately I noticed changelog.txt - which after a quick inspection confirmed the latest version of SweetRice was, in fact, 1.5.1. The same version that came back with multiple vulnerabilities.

    After digging through the rest of the pathways, I also found a SQL backup file - which I quickly downloaded and searched.

    I searched the file and found a hash digest and a username manager, So I ran hashcat --identify, which brought back MD4 & MD5 as the highest likelihood. After running both algorithms against rockyou.txt - I cracked the password.

    Remember: hashcat --identify isn't a determinate answer, it's the most likely answer from top down.

    I moved onto trying the Cross-Site Request Forgery / PHP Code Execution (#40700) - which is where we begin our direct exploitation of the system, gathering both flags.

    This exploit was a bit more complex. It involved injecting PHP wrapped commands into the Ads section of the CMS dashboard, then returning to the directory /content/inc/ads, loading the ad file in-browser (double-click), then hard reloading the ads directory which resulted in a new Ad file containing your command response (unless the command runs silently).

    Key Takeaways

    Version Enumeration via Changelog
    Always check for changelog.txt, readme, or version files during directory enumeration. They often reveal the exact software version and point you toward known exploits.

    Database Backups Are Gold
    SQL backup files are frequently left exposed. Always search for .sql, .bak, and backup files — they commonly contain plaintext credentials or password hashes.

    CMS Feature Abuse
    Once inside a CMS admin panel, explore every feature (especially content/ads/uploads). Many older CMS platforms allow direct PHP code injection through seemingly innocent fields.

    Sudo Script Analysis
    When you find a script you can run with sudo, always read it. Misconfigured scripts that call other files with weak permissions are a very common privilege escalation path.

    File Permission Enumeration
    World-writable files (especially ones executed by root via sudo or cron) are high-value targets. Always check permissions with ls -la on anything that looks important.
